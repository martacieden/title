# План впровадження нового підходу створення назви та опису

## Огляд
Ми переходимо від статичних полів "Назва" та "Опис" до динамічного процесу: вільний ввід ідеї -> AI покращення -> редагування результату.

## Кроки реалізації

### 1. Зміна станів у `CategoryModal`
- Додати `categoryIdea: string` для зберігання початкової ідеї.
- Додати `isEnhanced: boolean` для відстеження стану інтерфейсу.
- Додати `isEnhancing: boolean` для відображення лоадера під час роботи AI.

### 2. Оновлення інтерфейсу Step 1
- **Початковий стан (`!isEnhanced`)**:
  - Приховати поля `categoryName` та `categoryDescription`.
  - Додати поле `Idea` (Textarea) з міткою "What would you like to create?".
  - Додати кнопку "Enhance with AI".
- **Стан після покращення (`isEnhanced`)**:
  - Відобразити покращені дані у спеціальному контейнері-картці (як на скріншоті).
  - Додати поле `Title` (Input) всередині картки.
  - Додати поле `Description` (Textarea) всередині картки.
  - Додати кнопку "Re-enhance" у нижньому правому куті картки.
  - Додати кнопку "Edit Idea" для повернення до початкового стану.

### 3. Симуляція AI логіки
- Створити функцію `enhanceIdeaWithAI`, яка:
  - Включає лоадер.
  - Через 1.5-2 секунди розбиває `categoryIdea` на `title` та `description`.
  - Оновлює стани `categoryName`, `categoryDescription` та `isEnhanced`.

### 4. Покращення AI Промпту (для майбутнього)
- Наразі ми зробимо базову логіку розбиття, але підготуємо структуру для виклику реального API.
