# План покращень для Title Settings

## Аналіз поточного стану

### Що вже працює:
- ✅ Тогл вимкнений за замовчуванням (`templateEnabled = false`)
- ✅ Показуються 3 варіанти одразу (radio buttons)
- ✅ Можливість вибрати варіант
- ✅ Кнопка "Generate new options"

### Що потрібно виправити/додати:

## 1. Лоадер при включенні тоглу
**Проблема**: Коли користувач включає тогл, має бути лоадер, поки AI генерує контент.

**Рішення**:
- Додати стан `isGeneratingSuggestions`
- При включенні тоглу показувати skeleton/loader
- Після "генерації" (затримка ~1-2 сек) показувати варіанти

## 2. Покращити пояснення
**Проблема**: Треба краще пояснити, що генерує title автоматично з form data / custom fields.

**Рішення**:
- Оновити опис: "Generate titles automatically from form data and custom fields"
- Додати пояснення, як воно буде відображатися
- Можливо додати приклад з поясненням

## 3. Показати метадані/технічні деталі знизу
**Проблема**: Користувач не розуміє, як складається title, які поля використовуються.

**Рішення**:
- Показувати знизу (під варіантами) технічні деталі:
  - Які поля використовуються (System Fields / Custom Fields)
  - Як воно складається (template з поясненням)
- Показувати тільки для вибраного варіанту
- Або показувати завжди, але компактно

## 4. Прибрати редундантність
**Проблема**: RadioGroup і Edit template / Setup manually працюють однаково.

**Рішення**:
- Видалити "Edit template" з AI view
- Залишити тільки "Customize manually" внизу
- При переході в manual mode показувати вибраний template для редагування

## 5. Можливість редагувати вибраний варіант
**Проблема**: Після вибору варіанту немає можливості його відредагувати без переходу в manual mode.

**Рішення**:
- Додати кнопку "Edit" або "Customize" для вибраного варіанту
- Або додати можливість клікнути на вибраний варіант для редагування
- При редагуванні показувати template editor з вибраним значенням

## 6. Приховати технічні деталі до ручного редагування
**Проблема**: Технічні деталі (variables, template editor) показуються завжди в manual mode.

**Рішення**:
- В AI view показувати тільки варіанти з preview
- Технічні деталі (як складається) показувати компактно знизу
- Повний template editor показувати тільки при переході в manual mode
- В manual mode показувати вибраний template для редагування

## 7. Покращити UX порівняння варіантів
**Проблема**: Важко порівнювати варіанти, якщо треба все прокликати.

**Рішення**:
- ✅ Вже реалізовано: показуємо всі 3 варіанти одразу
- Додати можливість побачити template кожного варіанту (можливо при hover або кліку)
- Показувати preview з реальними даними (вже є)

## 8. Збереження вибраного варіанту
**Проблема**: При редагуванні потрібно зберігати вибраний варіант.

**Рішення**:
- При виборі варіанту зберігати його template в `templateValue`
- При переході в manual mode показувати вибраний template
- При поверненні з manual mode показувати вибраний варіант

## 9. Оновити текст кнопки "Generate new options"
**Проблема**: Можливо потрібно краще пояснити, що робить кнопка.

**Рішення**:
- Залишити як є, або змінити на "Get more options"

## 10. Покращити відображення вибраного варіанту
**Проблема**: Потрібно чітко показати, який варіант вибрано.

**Рішення**:
- ✅ Вже реалізовано: border-[#2563EB] і bg-[#EFF6FF]
- Можливо додати іконку або індикатор

## Пріоритети:

### Високий пріоритет:
1. Лоадер при включенні тоглу
2. Показати метадані/технічні деталі знизу
3. Можливість редагувати вибраний варіант
4. Приховати технічні деталі до ручного редагування

### Середній пріоритет:
5. Покращити пояснення
6. Прибрати редундантність

### Низький пріоритет:
7. Покращити UX порівняння варіантів (вже частково реалізовано)
8. Оновити текст кнопки

## Технічні деталі реалізації:

### Додати стани:
- `isGeneratingSuggestions: boolean` - для лоадера
- `selectedTemplateForEdit: string | null` - для редагування вибраного

### Компоненти:
- Skeleton/Loader компонент для генерації
- Compact metadata view під варіантами
- Template editor в manual mode з вибраним значенням

### Логіка:
- При включенні тоглу: показувати лоадер → генерувати → показувати варіанти
- При виборі варіанту: зберігати template, показувати метадані
- При кліку "Edit" на варіанті: переходити в manual mode з вибраним template
- В manual mode: показувати template editor з вибраним значенням
